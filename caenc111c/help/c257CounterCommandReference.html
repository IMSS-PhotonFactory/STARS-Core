About CAEN Module C257 counter commands

About CAEN Module C257 counter commands
CAEN Module C257 has 2 types of command system, controller command system and counter command system.
Counter command system is designed to specify the counters of CAEN Module C257 by Stars client name.
Each counter of CAEN Module C257 has differrent Stars client name, for example, 'caenc111c.C01' for CH0, 'caenc111c.C02' for CH1, ...'caenc111c.C16' for CH15.

How to execute the Stars Command of CAEN Module C257 counter

How to execute the Stars Command of CAEN Module C257 counter

Send the Stars message as following format

caenc111c.C01 CAEN_C257_Counter_Stars_Command Command_Parameters

  • 'caenc111c.C01' is the Stars client name for CAEN Module C257 counter CH0

  • CAEN_C257_Counter_Stars_Command

  • Command_Parameters    (Only necessary)


Then reply arrives as following

caenc111c.C01>test @CAEN_C257_Counter_Stars_Command Command_Parameters Command_Message

  • 'test' is used, in this manual, as the Stars client name that sends the Stars message.

  • Command_Message
    • starts with 'Ok: ' : the Stars command executed with success.

    • starts with 'Er: ' : the Stars command did not executed because of command nonexistent, command paramarer invalid, etc.

    • starts with 'Ng: ' : the Stars command executed but failed.

Error General

Error General

  1. Destination client name nonexistent :
    • (send)  caenc111c.c01 hello
      (reply)  System>test @hello Er: caenc111c.c01 is down.

  2. Command nonexistent :
    • (send)  caenc111c.C01 hell
      (reply)  caenc111c.C01>test @hell Er: Bad Command.

  3. Command parameter missing or invalid :
    • (send)  caenc111c.C01 GetValue 1
      (reply)  caenc111c.C01>test @GetValue 1Er: Bad Parameter.

  4. Command executed but network error occured :
    (Usually not happens. But if it's happened, stop 'caenc111c' Program, restart CAMAC Controller CAEN C111C. then start 'caenc111c')
    • (send)  caenc111c.C01 GetValue
      (reply)  caenc111c.C01>test @GetValueEr: -4

GetCounterNumber

GetCounterNumber
Returns the counter number of the selfnode.

[Usage]
caenc111c.c257 GetCounterNumber

[Reply Message Format]
@GetCounterNumber Ok: CounterNumber    (When command executed successfully)
  1. CounterNumber    0 - 15

[Examples]
(sender)
caenc111c.C02 GetCounterNumber
(Receive messages)
caenc111c.C02>test @GetCounterName Ok: 1

Counter Command (Access to CAEN Model C257 counter)

GetValue

GetValue
Returns a value held in the specified counter(0..15). [F(0) A(0-15)]

[Usage]
caenc111c.C01 GetValue

[Reply Message Format]
@GetValue Ok: Value    (When command executed successfully)
  1. Value    0 - 16777215

[Examples]
(sender)
caenc111c.C01 GetValue
(Receive messages)
caenc111c.C01>test @GetValue Ok: 7

GetValueInhibitReset

GetValueInhibitReset
Returns a value held in the specified counter(0..15), and when reads the 15th counter (last counter), if the input is inhibited, resets all the counters(0..15) and the LAM line. [F(2) A(0-15)]

[Usage]
caenc111c.C01 GetValueInhibitReset

[Reply Message Format]
@GetValueInhibitReset Ok: Value    (When command executed successfully)
  1. Value    0 - 16777215

[Examples]
(sender)
caenc111c.c257 GetInhibitIsBusy
caenc111c.C01 GetValueInhibitReset
caenc111c.C16 GetValueInhibitReset
caenc111c.C01 GetValueInhibitReset
(Receive messages)
caenc111c.c257>test @GetInhibitIsBusy Ok: 1
caenc111c.C01>test @GetValueInhibitReset Ok: 7
caenc111c.C16>test @GetValueInhibitReset Ok: 14
caenc111c.C01>test @GetValueInhibitReset Ok: 0
