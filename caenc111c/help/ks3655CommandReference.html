General

Stars client name of Kenetic System Model 3655-L1A

Stars client name of Kenetic System Model 3655-L1A
In this Manual, 'caenc111c.ks3655' is used for the client name of Kenetic System Model 3655-L1A.

How to execute the Stars Command of Kenetic System Model 3655-L1A

How to execute the Stars Command of Kenetic System Model 3655-L1A

Send the Stars message as following format

caenc111c.ks3655 KS3655_Stars_Command Command_Parameters

  • 'caenc111c.ks3655' is the Stars client name for Kenetic System Model 3655-L1A.

  • KS3655_Stars_Command

  • Command_Parameters    (Only necessary)


Then reply arrives as following

caenc111c.ks3655>test @KS3655_Stars_Command Command_Parameters Command_Message

  • 'test' is used, in this manual, as the Stars client name that sends the Stars message.

  • Command_Message
    • starts with 'Ok: ' : the Stars command executed with success.

    • starts with 'Er: ' : the Stars command did not executed because of command nonexistent, command paramarer invalid, etc.

    • starts with 'Ng: ' : the Stars command executed but failed.

Error General

Error General

  1. Destination client name nonexistent :
    • (send)  caenc111c.ks365 hello
      (reply)  System>test @hello Er: caenc111c.ks365 is down.

  2. Command nonexistent :
    • (send)  caenc111c.ks3655 hell
      (reply)  caenc111c>test @hell Er: Bad Command.

  3. Command parameter missing or invalid :
    • (send)  caenc111 SetDelayMultiplier 1
      (reply)  caenc111c>test @SetDelayMultiplier 1 Er: Bad Parameter.

  4. Command executed but network error occured :
    (Usually not happens. But if it's happened, stop 'caenc111c' Program, restart CAMAC Controller CAEN C111C. then start 'caenc111c')
    • (send)  caenc111c SetDelayMultiplier 0 1
      (reply)  caenc111c>test @SetDelayMultiplier 0 1Er: -4

SetCycleControlRegister

SetCycleControlRegister
Set the Control Register value to select timing source, the number of output channels and single cycle/recycle. [F(17) A(0)]

[Usage]
caenc111c.ks3655 SetCycleControlRegister TimingSource NumberOfChannels SingleCycleOrRecycle

  1. TimingSource

    • Select the internal Time Base
      • 0    1-SEC Time Base

      • 1    100m-SEC Time Base

      • 2    10m-SEC Time Base

      • 3    1m-SEC Time Base

      • 4    100micro-SEC Time Base

      • 5    10micro-SEC Time Base

      • 6    1micro-SEC Time Base

    • Select the external clock input
      • 7    external clock

  2. NumberOfChannels
    • 0    Output Channel 1. t1 only
    • 1    Output Channel 1 and 2. t1-t2
    • 2    Output Channel 1 to 3. t1-t3
    • 3    Output Channel 1 to 4. t1-t4
    • 4    Output Channel 1 to 5. t1-t5
    • 5    Output Channel 1 to 6. t1-t6
    • 6    Output Channel 1 to 7. t1-t7
    • 7    Output Channel 1 to 8. t1-t8

  3. SingleCycleOrRecycle    
    • 0    Single Cycle
    • 1    Recycle

[Reply Message Format]
@SetCycleControlRegister TimingSource NumberOfChannels SingleCycleOrRecycle Ok:    (When command executed successfully)
@SetCycleControlRegister TimingSource NumberOfChannels SingleCycleOrRecycle Er: Bad Parameter.    (When any of specified parameters invalid)

[Examples]
(sender)
caenc111c.ks3655 SetCycleControlRegister 0 1 0    (1-SEC Time Base, Output t1, Single Cycle)
(Receive messages)
caenc111c.ks3655>test @SetCycleControlRegister 0 1 0 Ok:

SetDelayMultiplier

SetDelayMultiplier
Set the multiplier value for each channel which is used for calculating timing delay after t0. [F(16) A(0-7)]
If 1-SEC Time Base selected (Show command 'SetCycleControlRegister'), and set the multiplier for Channel 1 is 20 (decimal), the Channel 1 pulse will occur 20 seconds after t0.
If the Channel 2 set the multiplier is 30, it's pulse will occur 30 seconds after t0, and 10 seconds after t1.

[Usage]
caenc111c.ks3655 SetDelayMultiplier ChannelNumber Multiplier

  1. ChannelNumber
    • 0    Set the multiplier for Channel 1
    • 1    Set the multiplier for Channel 2
    • 2    Set the multiplier for Channel 3
    • 3    Set the multiplier for Channel 4
    • 4    Set the multiplier for Channel 5
    • 5    Set the multiplier for Channel 6
    • 6    Set the multiplier for Channel 7
    • 7    Set the multiplier for Channel 8

  2. Multiplier    1 - 65535 (decimal)

[Reply Message Format]
@SetDelayMultiplier ChannelNumber Multiplier Ok:    (When command executed successfully)
@SetDelayMultiplier ChannelNumber Multiplier Er: Bad Parameter.    (When any of specified parameters invalid)

[Examples]
(sender)
caenc111c.ks3655 SetDelayMultiplier 0 1
(Receive messages)
caenc111c.ks3655>test @SetDelayMultiplier 0 1 Ok:

SetInhibitEnable

SetInhibitEnable
Enables or Disables of the ability of the module, that can be set to assert the Dataway Inhibit line coincident with any output pulse, release that line coincident with another output pulse. [F(24 or 26) A(9)]
This command needs to be run after power-up or initialize.
If the Dataway Inhibit line has been enabled, next run command 'SetInhibitRegister' to set the channel number to set inhibit and to set the another number to clear inhibit.

[Usage]
caenc111c.ks3655 SetInhibitEnable 0 | 1
  • 0   Set disable
  • 1   Set enable

[Reply Message Format]
@SetInhibitEnable 1 Ok:    (Enabled)
@SetInhibitEnable 0 Ok:    (Disabled)
@SetInhibitEnable other Er: Bad Parameter.     (When a specified parameter invalid)

[Examples]
(sender)
caenc111c.ks3655 SetInhibitEnable 1
(Receive messages)
caenc111c.ks3655>test @SetInhibitEnable 1 Ok:

    • 7    Set inhibit coincident with output Channel 8, t8

  2. ChannelNumberToClearInhibit

    • 0    Set clear inhibit coincident with output Channel 1, t1

    • 3    Set clear inhibit coincident with output Channel 4, t4

SetLAMMaskRegister

SetLAMMaskRegister
Set the LAM Mask Register value to assert a LAM when a particular channel is pulsed. [F(17) A(13)]

[Usage]
caenc111c.ks3655 SetLAMMaskRegister Value

  1. Value    0 - 255 (decimal)
        Value consists of 8-Bit. Bit position(1,2..8) is used to select the channel number, and bit value '1' is used to set LAM on.

    • 0     NO LAM

    • 1     LAM when output Channel 1 (t1) is pulsed.

    • 2     LAM when output Channel 2 (t2) is pulsed.

    • 4     LAM when output Channel 3 (t3) is pulsed.

    • 8     LAM when output Channel 4 (t4) is pulsed.

    • 16     LAM when output Channel 5 (t5) is pulsed.

    • 32     LAM when output Channel 6 (t6) is pulsed.

    • 64     LAM when output Channel 7 (t7) is pulsed.

    • 128     LAM when output Channel 8 (t8) is pulsed.



[Reply Message Format]
@SetLAMMaskRegister Value Ok    (When command executed successfully)
@SetLAMMaskRegister Value Er: Bad Parameter.    (When a specified parameter invalid)

[Examples]
(sender)
caenc111c.ks3655 SetLAMMaskRegister 2    (LAM when output Channel 2 (t2) is pulsed.)
(Receive messages)
caenc111c.ks3655>test SetLAMMaskRegister 2 Ok:

GetValueLAMStatusBits

GetValueLAMStatusBits
Returns a current decimal value of LAM status bits(8-bit). [F(1) A(12)]

[Usage]
caenc111c.ks3655 GetValueLAMStatusBits

[Reply Message Format]
@GetValueLAMStatusBits Ok: Value    (When command executed successfully)

  1. Value    0 - 255 (decimal)
        Value consists of 8-Bit. Bit position(1,2..8) is used to show the channel number, and bit value '1' shows that a particular channel has been pulsed.

    • 0     When No output Channel has been pulsed.

    • 1     When output Channel 1 (t1) has been pulsed.

    • 3     When output Channel 1 and 2 (t1,t2) has been pulsed.

    • 7     When output Channel 1 to 3 (t1-t3) has been pulsed.

    • 15     When output Channel 1 to 4 (t1-t4) has been pulsed.

    • 31     When output Channel 1 to 5 (t1-t5) has been pulsed.

    • 63     When output Channel 1 to 6 (t1-t6) has been pulsed.

    • 127     When output Channel 1 to 7 (t1-t7) has been pulsed.

    • 255     When output Channel 1 to 8 (t1-t8) has been pulsed.

[Examples]
(sender)
caenc111c.ks3655 GetValueLAMStatusBits
(Receive messages)
caenc111c.ks3655>test @GetValueLAMStatusBits Ok: 3    (when output Channel 1(t1) and 2(t2) has been pulsed.)
